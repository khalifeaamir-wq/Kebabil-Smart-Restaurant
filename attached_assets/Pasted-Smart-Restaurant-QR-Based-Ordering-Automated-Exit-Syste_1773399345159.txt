Smart Restaurant QR-Based Ordering & Automated Exit System

The solution must be modern, scalable, mobile-first, operationally realistic, and polished enough to be used as a final year project with real commercial potential.

The system should eliminate traditional bottlenecks in restaurant operations by combining:

QR-based table ordering

Live kitchen workflow management

Digital payment

Temporary exit authorization QR

Smart door unlock validation

Analytics and restaurant operations dashboard

1. Core Product Vision

Design and build a restaurant platform where the customer can complete the full dining flow from their phone:

Scan the QR on the table

View a digital menu linked to that exact table

Add items to cart and place an order

Kitchen receives the order instantly

Staff prepares and serves to the correct table

Customer pays digitally from phone

System generates a short-lived exit QR code

Customer scans at the exit scanner

Smart door unlocks only for validated paid sessions

This system must reduce:

manual ordering delays

billing counter congestion

waiter dependency for basic flows

unpaid exits

table-service confusion

3. Required User Flows
Flow A — Customer Enters and Scans Table QR

Each table has a permanent unique QR code such as:

restaurant.com/table/12

When scanned:

system identifies the table number automatically

opens the customer menu page

stores session mapped to the table

marks table as occupied if no active dining session exists

Flow B — Ordering

Customer can:

browse categories

see dish image, name, description, price, spice level, veg/non-veg tag

customize quantity

add notes for kitchen

add items to cart

place order

After placing order:

generate order ID

show estimated preparation time

push order to kitchen dashboard

show live order status on customer side

Flow C — Kitchen Handling

Kitchen dashboard must show:

order number

table number

ordered items

quantity

special notes

timestamp

preparation timer

current status

Kitchen statuses:

New

Accepted

Preparing

Ready

Served

When status changes:

customer should see live updates

waiter dashboard should receive service notifications when status becomes “Ready”

Flow D — Payment

Customer should be able to click:

Pay Now
or

Scan table QR again and reopen current table session

Bill must show:

ordered items

quantity

subtotal

taxes

grand total

payment status

Supported payments:

UPI

Card

Wallet

Cash toggle for admin/manual completion if needed

After successful digital payment:

mark order/session as paid

close pending payment state

generate temporary exit QR

Flow E — Exit QR

After payment, generate a secure temporary QR code with short validity such as 3 minutes.

The QR must not be a plain editable value. It must be:

signed

time-bound

single-use if possible

linked to a paid session

Example payload logic:

{
  "session_id": "abc123",
  "order_id": 231,
  "table_id": 12,
  "paid": true,
  "issued_at": "timestamp",
  "expires_at": "timestamp",
  "token_signature": "secure_hash"
}
Flow F — Smart Door Access

At exit:

QR scanner scans temporary exit QR

backend verifies token

checks payment status

checks expiration time

checks whether token already used

if valid, triggers smart lock unlock signal

log access event in database

If invalid:

door remains locked

show clear failure reason on admin/guard interface
